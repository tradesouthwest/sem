SEM - Semantic Element Modeling

Utilizing HTML5 tags and elements to create a document, without any divs or containers

SEM is a trial markup ideology which takes advantage of the Semantic Elements and use them as markup to create a fully supported Browser document.

See: HTML5 Semantic Elements for difference between HTML5 Semantic Elements and NON HTML5 Semantic Elements

In the HTML5 standard, the <article> element defines a complete, self-contained block of related elements. The <section> element is defined as a block of related elements. SEM adds a bit of styling attributes to the Elements and turns these elements into selectors of their own.

I do not intend on rebuilding the wheel by making the Elements completely a "framework-ready rework" of HTML5, this experiment is only to allow minimal amount of styling so as to not throw off any DOM relationships the Semantic Elements have with any supported Browser. The only real intention is to relieve the programmer from the use of the <div> tag and apply style attributes directly to the Semantic elements.

Please Note that this practice is not recommended for any professional production level Website. You may pass W3C Validation but it is not guaranteed that all associated markup of the 'SEM' style will be universally accepted.
